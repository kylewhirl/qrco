The tqrco API is a versioned REST surface for managing QR codes and the data around them.

## Base URL

During local development:

```bash
http://localhost:3002
```

In production today:

```bash
https://tqrco.de
```

All endpoints are versioned under `/api/v1`.

## Authentication

Use a bearer API key generated from the dashboard settings page.

```bash
curl https://tqrco.de/api/v1/qr-codes \
  -H "Authorization: Bearer tqr_live_..."
```

Every API key is scoped to the Stack user that created it. The API never hands the key direct database access. Each request resolves the key to a user id and then applies ownership filters in the query layer.

## Response envelope

Most successful JSON endpoints return a `data` object or array:

```json
{
  "data": []
}
```

Mutation-style delete endpoints may return:

```json
{
  "success": true
}
```

## Image rendering

Saved QR codes can be rendered directly as SVG or PNG:

```bash
curl -L \
  -H "Authorization: Bearer tqr_live_..." \
  "https://tqrco.de/api/v1/qr-codes/QR_ID/render?format=svg"
```
